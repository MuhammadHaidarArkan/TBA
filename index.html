Hasil Konstruksi PDA

Penjelasan Konfigurasi 7-Tuple PDA

Karena kita menggunakan metode Top-Down Parsing, logika utama dijalankan di dalam Stack, sehingga kita hanya butuh 3 state: q1 untuk persiapan, q2 untuk proses penjabaran & pencocokan berulang, dan q3 untuk penerimaan akhir.

Σ (Alfabet Input) = { {{ pdaCfgResult.Sig.join(', ') }} }

Merupakan kumpulan seluruh simbol Terminal (huruf kecil) yang dihasilkan oleh tata bahasa. Simbol ini yang akan dicocokkan dengan pita input dari user.

Ini adalah semua simbol yang boleh dimasukkan ke dalam Stack. Terdiri dari Terminal, Variabel (karena kita akan menjabarkannya di stack), ditambah Z0 sebagai penanda batas terbawah stack.

S (State Awal) = q1

Titik awal jalannya mesin sebelum memproses input apapun.

Z0 (Top Stack Awal) = Z0

Sebelum string diperiksa, stack tidak sepenuhnya kosong melainkan diisi oleh pondasi penanda Z0.

F (Himpunan State Akhir) = { q3 }

Untai (string) akan dinyatakan Valid/Diterima jika dan hanya jika mesin berhasil mencapai state penyelesaian ini.

Konfigurasi PDA Ekuivalen Baru (M2)

Kita mempertahankan semua state dari mesin lama, namun menambahkan qs sebagai state persiapan dan qf sebagai state penerimaan spesifik yang baru.

Semua alfabet stack lama dipertahankan, ditambah simbol baru X yang berfungsi sebagai pelindung atau alarm batas bawah stack.

Mesin diwajibkan selalu mulai dari state persiapan khusus (qs) agar pondasi pelindung Stack bisa dibangun lebih dulu.

Z'0 (Dasar Stack Baru) = X

Lantai dasar stack bukan lagi Z0 milik mesin asli, melainkan simbol proteksi X yang kita ciptakan.

F' = { }

Karena mesin baru ini berbasis Final State, kita menetapkan qf sebagai satu-satunya tujuan akhir penyelesaian.

Mesin Null Stack tidak memiliki Final State. Keberhasilan murni diukur dari habisnya isi Stack di akhir pemrosesan.
